BOFH

Browser-native dev toolkit. Nothing leaves your device.

What's in here

  • JSON — format, validate, tree-view, JSONPath
  • JWT — decode header/payload, verify HS256/RS256, check expiry
  • Base64 — encode/decode text and files, URL-safe variant
  • Timestamp — Unix epoch ↔ human, timezone-aware, relative
  • UUID — generate v4 / v7, bulk, validate

More modules coming. Every transformation runs locally — no servers, no telemetry, no random S3 buckets seeing your tokens.

Help & shortcuts

Keyboard

  • / — focus the module filter
  • ? — open this help
  • Esc — close modal / blur input

State

Each module saves its last input and settings to localStorage under namespaced keys (bofh:<module>:…). Use the ⌫ button in the sidebar foot to wipe everything.

Linkable tabs

Each module has a hash URL (#json, #jwt, …) so you can bookmark or share a tab.

JSON

Paste or drop JSON. Format, minify, validate, or browse as a collapsible tree. JSONPath (subset) supports $, dot/bracket paths, and array indices.

JWT

Decode any JWT. Verify HS256 by typing the secret. Verify RS256 by pasting the issuer's public key in PEM (-----BEGIN PUBLIC KEY-----). Expiry and "not before" are checked automatically.

Base64

Text mode encodes UTF-8. File mode reads any file as bytes. URL-safe toggle uses -_ instead of +/ and strips padding.

Timestamp

Detects seconds vs milliseconds. Shows ISO, local, and a relative ("3 days ago") form. Timezone selector uses your browser's IANA list.

UUID

v4 is fully random. v7 is time-ordered (UNIX-ms timestamp + random) and database-friendly. Bulk-generate up to 1000.

BOFH is a single HTML file. There is no server, no analytics, no fonts loaded from a CDN, no third-party scripts (Phase 1).

Your inputs are stored only in your own browser's localStorage. Clear it with the ⌫ button or your browser's site-data settings.

The JWT verifier uses the browser's built-in Web Crypto API — the same primitives your OS uses.
